# Retro FPS Toolkit (RFK) - 2.5D Modern Sector Based Game Engine

**Кроссплатформенный 3D движок в стиле шутеров 90-х на Go**

> Секторальная геометрия (Doom-like), современный рендеринг на OpenGL 4.1 и полный низкоуровневый контроль над графическим пайплайном.

Этот инструмент - основа для создания быстрых, атмосферных шутеров с современной графикой и классической логикой построения уровней.

## 🚀 Основные особенности

- **Дробление логики**

  - Фундамент проекта (framework) находится в пакетах `core`, `render` и `app`
  - Пакет `app` выступает оркестратором и "поставщиком услуг". Пакет предоставляет следующие интерфейсы:

    - **App API**

      Этот интерфейс определяет набор системных услуг и данных, которые «Оркестратор» предоставляет активному состоянию игры. Он служит мостом, позволяя состояниям взаимодействовать с низкоуровневыми подсистемами, не владея ими напрямую.

    - **App State**

      Этот интерфейс представляет собой логическую единицу приложения (например, главное меню, уровень игры или редактор). Он управляет собственным жизненным циклом и обрабатывает события ввода.

- **Низкоуровневый контроль над высокоуровневыми инструментами**

  - Базовая комплектация RFK предоставляет возможность контроля за внутренним состоянием _OpenGL_ и существующим конвеером рендеринга (выделением VRAM, создание FBO)
  - Пользователь имеет возможность создать свой конвеер или использовать интерфейс `AppAPI` (RenderGBuffer, RenderFlat)

> **Движку насрать что рендерить** - хоть 2.5D шутер, хоть научная визуализация. Он предоставляет базовый пакет услуг. Вы можете \*\*удалить пакет `kit` из `pkg` и написать свой функционал - RFK переварит!

## 🛠 Стек технологий

- **Язык:** Go (Golang)
- **Графика:** OpenGL 4.1 (core profile)
- **Библиотеки:**
  - `github.com/go-gl/gl/v4.1-core/gl` — привязки OpenGL
  - `github.com/go-gl/glfw/v3.3/glfw` — создание окна, ввод
  - `github.com/AllenDang/cimgui-go` — ImGui (инструменты разработчика)
  - `github.com/go-gl/mathgl/mgl64` — математика (матрицы, векторы)

## 📂 Структура проекта

```
.
├── cmd
│   └── rfk
│          rfk.go            # Точка входа
│
├── pkg/
│   ├── app/                 # Оркестратор приложения и его "свита"
│   ├── core/                # Автономный пакет, базовые примитивы и логика
│   ├── render/              # Отображение (как рисовать)
│   └── toolkit/             # Базовый набор инструментов
│       ├── assets/          # Загрузка данных и модели (LevelLoader, LevelModel)
│       ├── systems/         # Системы (Level / LevelRenderer)
│       ├── entities/        # Описания сущностей (Player, NPC, GUI Text, Particle)
│       └── states/          # Игровые состояния (главное меню, редактор, игровой уровень)
│
└── assets/                  # Текстуры, модели, шейдеры, блюпринты
```

Фактически вся суть разработки на **RFK** сводится к написанию своих состояний. Приложение предоставляет **AppAPI**, как интерфейс взаимодействия с базовым функционалом.

## 🔧 Установка и сборка

### Требования

- **Go** 1.25 или выше
- **OpenGL 4.1** совместимая видеокарта

### 🛠️ Сборка

```bash
git clone https://github.com/illoprin/retro-fps-kit-go.git
cd retro-fps-kit-go
go mod tidy
go build -ldflags="-H windowsgui -s -w" -o rfk.exe .\cmd\rfk\rfk.go
```

`-H windowsgui` - создает GUI-приложение

`-s` - удаляет таблицу символов

`-w` - удаляет отладочную информацию DWARF
